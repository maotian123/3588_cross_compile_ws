# 从零构建 RK3588 交叉编译环境完整指南

本文档描述从一台已配置好的 RK3588 板子，构建宿主机交叉编译环境的完整流程。

## 0. 总览

```text
RK3588 板端确认
  -> x86 宿主机安装 aarch64 交叉编译器和工具
  -> 从 RK3588 抽取 sysroot
  -> 解压 sysroot 并生成 cross_compile_env.sh
  -> 链接或 clone loc_map
  -> CMake toolchain 交叉编译
  -> 本地 file 扫描
  -> 板端 /tmp 运行验证
```

本地调试流程不依赖 Docker。交叉编译隔离靠：

- `cmake/toolchain.cmake`
- `sysroot_base/`
- `cross_compile_env.sh`
- `install/`

CI/CD 场景使用 Docker 大镜像固定 x86 编译工具和 RK3588 sysroot，见 [RK3588_CI_Docker_交叉编译指南.md](RK3588_CI_Docker_交叉编译指南.md)。

## 2. 准备宿主机

宿主机安装工具：

```bash
sudo apt-get update
sudo apt-get install -y \
  build-essential \
  cmake \
  make \
  pkg-config \
  rsync \
  python3 \
  python3-pip \
  gcc-aarch64-linux-gnu \
  g++-aarch64-linux-gnu \
  qtbase5-dev-tools

python3 -m pip install --user paramiko
```

验证：

```bash
aarch64-linux-gnu-gcc --version
aarch64-linux-gnu-g++ --version
python3 - <<'PY'
import paramiko
print("paramiko ok")
PY
which moc uic rcc
```

工具链原则：

- 目标板 Ubuntu 20.04 / GCC 9.4 / glibc 2.31。
- 宿主机使用 Ubuntu Focal 系列 `aarch64-linux-gnu` GCC 9.x。
- 不用 Orin/L4T Bootlin GCC 9.3 作为 RK3588 默认工具链。

## 4. 抽取 sysroot

```bash
cd /home/user/3588_cross_compile_ws
RK3588_PASSWORD=<password> make extract-sysroot
```

输出：

```text
arch/rk3588_sysroot.tar.gz
```

抽取脚本入口：

```text
script/extract_sysroot.sh
script/rk3588_extract_sysroot.py
```

`RK3588_PASSWORD` 只在命令行临时传入，不写进文件。

## 5. 解压 sysroot 并生成环境

```bash
cd /home/user/3588_cross_compile_ws
make setup-sysroot
```

生成：

```text
sysroot_base/
cross_compile_env.sh
```

`cross_compile_env.sh` 设置：

```bash
RK3588_CROSS_COMPILE_SYSROOT=/home/user/3588_cross_compile_ws/sysroot_base
RK3588_WORKSPACE_DIR=/home/user/3588_cross_compile_ws
CMAKE_PREFIX_PATH=/home/user/3588_cross_compile_ws/install:...
CATKIN_PREFIX_PATH=/home/user/3588_cross_compile_ws/sysroot_base/opt/ros/noetic
PKG_CONFIG_SYSROOT_DIR=/home/user/3588_cross_compile_ws/sysroot_base
```

## 7. 检查前置条件


## 8. 编译

```bash
cd /home/user/3588_cross_compile_ws
source ./cross_compile_env.sh
RK3588_BUILD_JOBS=1 ./script/compile.sh
```

编译入口按顺序处理：

```text
LocUtils -> msf_loc -> slam_ui
```

单包：

```bash
./script/compile.sh --package LocUtils
./script/compile.sh --package msf_loc
./script/compile.sh --package slam_ui
```

```bash
find install/lib -maxdepth 1 -type f -name '*.so*' -exec file {} + | rg 'x86-64|Intel' || true

  -type f -exec file {} + | rg 'x86-64|Intel' || true
```

## 12. 上传前检查

```bash
cd /home/user/3588_cross_compile_ws
git status --short --ignored
git ls-files | rg '(__pycache__|\\.pyc$|^build/|^install/|^sysroot_base/|^arch/|^toolchain|^src/)'
```

第二条命令应无输出。若有输出，说明生成文件被跟踪，需要从 git index 移除。
