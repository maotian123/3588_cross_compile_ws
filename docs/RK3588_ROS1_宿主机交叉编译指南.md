# RK3588 ROS1 Noetic 宿主机交叉编译指南

本文档是日常使用入口：在 x86_64 宿主机上直接编译 RK3588/aarch64 的 `LocUtils`、`msf_loc` 和 `slam_ui`。

## 前提条件

### 目标平台

| 项目 | 值 |
|------|-----|
| 目标板 | Rockchip RK3588 CORE LP4 V10 Board |
| SSH | `sfzt@10.10.10.50` |
| OS | Ubuntu 20.04.6 LTS |
| 架构 | aarch64 / arm64 |
| Kernel | 5.10.160 |
| ROS | ROS1 Noetic, `/opt/ros/noetic` |
| GCC/G++ | 9.4.0 |
| glibc | 2.31 |
| Docker | 未安装 |

详细版本见 [RK3588_target_info.md](RK3588_target_info.md)。

### 宿主机

宿主机是 x86_64 Linux。本地调试可以不使用 Docker；CI/CD 正式路径使用带 RK3588 sysroot 的 Docker 大镜像，见 [RK3588_CI_Docker_交叉编译指南.md](RK3588_CI_Docker_交叉编译指南.md)。

本地直接编译需要这些工具：

```bash
sudo apt-get update
sudo apt-get install -y \
  build-essential \
  cmake \
  make \
  pkg-config \
  rsync \
  python3 \
  python3-pip \
  gcc-aarch64-linux-gnu \
  g++-aarch64-linux-gnu \
  qtbase5-dev-tools

python3 -m pip install --user paramiko
```

`paramiko` 用于从板端 SSH 抽取 sysroot。`moc`、`uic`、`rcc` 来自宿主机 Qt 工具，`slam_ui` 交叉编译时会执行这些 x86 工具，不能执行 sysroot 里的 ARM Qt 工具。

## 工具链选择

本工作区默认使用 Ubuntu Focal generic `aarch64-linux-gnu` GCC 9.x：

```bash
which aarch64-linux-gnu-gcc
aarch64-linux-gnu-gcc --version
```

选择原因：

- RK3588 板端是 Ubuntu 20.04 / GCC 9.4 / glibc 2.31。
- 该工具链来自 Ubuntu Focal 交叉编译包，目标 ABI 和板端系统匹配。
- NVIDIA Bootlin GCC 9.3 是 Orin/L4T 流程中的工具链，不作为 Rockchip RK3588 默认值。

## 提取和解压 sysroot

如果 `arch/rk3588_sysroot.tar.gz` 和 `sysroot_base/` 已存在，可以跳过提取，直接检查或构建。

重新从板端提取：

```bash
cd /home/user/3588_cross_compile_ws
RK3588_PASSWORD=<password> make extract-sysroot
make setup-sysroot
```

`make setup-sysroot` 会：

1. 解压 `arch/rk3588_sysroot.tar.gz` 到 `sysroot_base/`
2. 修复 sysroot 内绝对软链接
3. 生成 `cross_compile_env.sh`

密码不要写进文档、脚本或 git。

## 完整编译

```bash
cd /home/user/3588_cross_compile_ws
source ./cross_compile_env.sh
RK3588_BUILD_JOBS=1 ./script/compile.sh
```

等价 Makefile 入口：

```bash
cd /home/user/3588_cross_compile_ws
RK3588_BUILD_JOBS=1 make build
```

建议先用 `RK3588_BUILD_JOBS=1`，避免链接阶段内存压力导致随机失败。稳定后再调大。

构建顺序：

```text
LocUtils -> msf_loc -> slam_ui
```

## 单包编译

```bash
cd /home/user/3588_cross_compile_ws
source ./cross_compile_env.sh

./script/compile.sh --package LocUtils
./script/compile.sh --package msf_loc
./script/compile.sh --package slam_ui
```

也可以只构建某个 target：

```bash
./script/compile.sh --package msf_loc --target web_ui_websocket_node
```

注意：`msf_loc` 依赖 `LocUtils_DIR=$WORKSPACE/install/share/LocUtils`，所以 clean 后必须先编译并安装 `LocUtils`。

## 常见问题

### CMake 误用宿主机库

症状：`file` 扫描发现 `x86-64` 或 `Intel`。

处理：

```bash
find install/ -type f -exec file {} + | rg 'x86-64|Intel'
```

重点检查包内 `install()`、`find_package()`、`link_directories()` 是否把 `/usr/local/lib` 或宿主机库带进来了。

### Qt autogen 执行 ARM 工具失败

典型错误：

```text
/lib/ld-linux-aarch64.so.1: No such file or directory
```

原因是在 x86 宿主机上执行了 sysroot 内 ARM 版本 `moc/uic/rcc`。当前 `slam_ui` 已改为交叉编译时使用宿主机 `/usr/bin/moc`、`/usr/bin/uic`、`/usr/bin/rcc`。

### ROS/PCL CMake 路径重复拼接

重新运行修复脚本：

```bash
source /home/user/3588_cross_compile_ws/cross_compile_env.sh
/home/user/3588_cross_compile_ws/script/fix_pcl_ros_paths.sh \
  "$RK3588_CROSS_COMPILE_SYSROOT"
```

脚本带版本标记，必要时先删除 `sysroot_base/.cmake_pkgconfig_paths_fixed*` 再重跑。
